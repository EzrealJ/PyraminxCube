# RBAC 功能设计文档

> 本文档列出 RBAC 权限管理系统的完整功能清单，作为开发实现的依据

---

## 功能模块总览

```
┌─────────────────────────────────────────────────────────────────┐
│                      RBAC 功能模块                              │
├─────────────────────────────────────────────────────────────────┤
│  1. 用户管理          │  用户的增删改查、状态管理、角色分配      │
│  2. 角色管理          │  角色的增删改查、权限分配                │
│  3. API权限管理       │  API的注册、分组、角色授权               │
│  4. 功能按钮权限管理   │  功能树维护、按钮与API关联               │
│  5. 数据权限管理       │  维度定义、维度值管理、角色数据权限配置  │
│  6. 权限校验服务       │  API拦截、数据过滤                      │
│  7. 系统管理          │  日志审计、缓存管理                      │
└─────────────────────────────────────────────────────────────────┘
```

## 1. 用户管理

### 1.1 功能清单

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|---------|---------|---------|-------|
| U-001 | 用户列表 | 分页查询用户，支持按用户名、邮箱、状态筛选 | P0 |
| U-002 | 新增用户 | 创建新用户，填写基本信息 | P0 |
| U-003 | 编辑用户 | 修改用户基本信息 | P0 |
| U-004 | 删除用户 | 软删除用户（标记为无效） | P0 |
| U-005 | 启用/禁用用户 | 切换用户状态 | P0 |
| U-006 | 重置密码 | 管理员重置用户密码 | P1 |
| U-007 | 分配角色 | 为用户分配一个或多个角色 | P0 |
| U-008 | 查看用户权限 | 查看用户的有效权限（合并所有角色后） | P1 |

### 1.2 功能详细说明

#### U-001 用户列表

**功能描述**：管理员可以查看系统中所有用户，支持分页和筛选。

**筛选条件**：
- 用户名（模糊匹配）
- 邮箱（模糊匹配）
- 状态（启用/禁用/全部）
- 创建时间范围

**列表字段**：
| 字段 | 说明 |
|------|------|
| 用户名 | 登录账号 |
| 邮箱 | 用户邮箱 |
| 状态 | 启用/禁用 |
| 角色 | 已分配的角色（多个用逗号分隔） |
| 创建时间 | 账号创建时间 |
| 操作 | 编辑、删除、分配角色 |

---

#### U-007 分配角色

**功能描述**：为用户分配角色，一个用户可以拥有多个角色。

**交互方式**：穿梭框（Transfer）
- 左侧：未分配的角色
- 右侧：已分配的角色

**业务规则**：
- 用户至少保留一个角色（可选，根据业务需求）
- 角色变更后，用户权限立即生效（下次请求时）
- 需要刷新用户的权限缓存

### 2.2 功能详细说明

#### R-005 分配API权限

**功能描述**：为角色分配可访问的API接口权限。

**交互方式**：
- 按API分组展示，支持展开/收起
- 勾选分组 = 勾选该分组下所有API
- 支持单独勾选某个API

**界面布局**：
```
┌─────────────────────────────────────────────────────────┐
│  API权限分配                                            │
├─────────────────────────────────────────────────────────┤
│  □ 用户管理                                             │
│    ├─ ☑ GET /api/users - 获取用户列表                  │
│    ├─ ☑ POST /api/users - 创建用户                     │
│    ├─ □ PUT /api/users/{id} - 更新用户                 │
│    └─ □ DELETE /api/users/{id} - 删除用户              │
│                                                         │
│  □ 角色管理                                             │
│    ├─ □ GET /api/roles - 获取角色列表                  │
│    └─ ...                                              │
└─────────────────────────────────────────────────────────┘
```

**业务规则**：
- 精确授权：勾选哪个就授权哪个
- 支持按分组批量勾选
- 保存后立即生效

**交互方式**：树形勾选（Tree with Checkbox）

**界面布局**：
```

│    └─ □ 权限管理                                       │
```

**业务规则**：
- 精确授权：勾选哪个节点就授权哪个，不自动继承子级
- 勾选父节点时，提示是否同时勾选所有子节点（可选）

## 6. 权限校验服务

### 6.1 功能清单

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|---------|---------|---------|-------|
| V-001 | API权限校验 | 拦截请求，验证用户是否有API访问权限 | P0 |
| V-002 | 数据权限过滤 | 提供显式调用的数据过滤方法 | P0 |
| V-003 | 获取当前用户权限 | 返回用户的所有权限信息 | P0 |
| V-004 | 权限缓存管理 | 缓存用户权限，提升性能 | P1 |
| V-005 | 刷新用户权限缓存 | 权限变更后刷新缓存 | P1 |

### 6.2 功能详细说明

#### V-001 API权限校验

**功能描述**：自动拦截所有API请求，校验用户是否有访问权限。

**校验流程**：
```
请求进入
    ↓
检查是否需要校验（排除白名单）
    ↓
获取当前用户信息
    ↓
获取用户的API权限列表（从缓存或数据库）
    ↓
判断请求的API是否在权限列表中
    ↓
├─ 有权限 → 放行
└─ 无权限 → 返回 403 Forbidden
```

**白名单**：
- 登录接口
- 公开接口
- 健康检查接口

---

#### V-002 数据权限过滤

**功能描述**：提供显式调用的数据过滤方法。

**核心原则**：
- **显式调用**：开发者主动调用 `.WithDataScope()` 才过滤
- **代码可读**：一眼看出哪里控制了数据权限
- **灵活自由**：不调用就不过滤

**使用示例**：
```csharp
// 应用数据权限过滤
var orders = await _dbContext.Orders
    .WithDataScope()  // 显式调用
    .Where(o => o.Status == "pending")
    .ToListAsync();

// 不需要数据权限的场景
var allOrders = await _dbContext.Orders.ToListAsync();
```

---

#### V-003 获取当前用户权限

**功能描述**：提供API，返回当前登录用户的所有权限信息。

**返回数据**：
```json
{
  "apiPermissions": ["user:list", "user:create", "role:list"],
  "featurePermissions": ["system:user", "system:user:add", "system:role"],
  "dataScopes": {
    "DEPARTMENT": ["DEPT_RD", "DEPT_RD_FE"],
    "REGION": ["ALL"]
  }
}
```

**用途**：
- 前端获取权限后渲染界面
- 控制按钮显示/隐藏
- 控制菜单显示
